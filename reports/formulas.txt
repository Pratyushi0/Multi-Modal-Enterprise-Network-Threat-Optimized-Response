Step 1: Contextual Rolling EWMA Baseline (Pillar 1)
Before anomalies can be detected, the system must transform raw security logs into an adaptive behavioral baseline. Instead of using a static mean, we employ an Exponentially Weighted Moving Average (EWMA) to capture "low and slow" behavioral drift while accounting for daily individual human variance.
The Equations
To establish the baseline, we track the historical moving mean $\mu_{t}(x)$ and the tracking moving variance $\sigma^2_{t}(x)$ across a specified window span (configured at $\text{span} = 14$ days):
$$\lambda = \frac{2}{\text{span} + 1}$$
$$\mu_{t}(x) = \lambda x_t + (1 - \lambda)\mu_{t-1}(x)$$
$$\sigma^2_{t}(x) = (1 - \lambda)\left(\sigma^2_{t-1}(x) + \lambda(x_t - \mu_{t-1}(x))^2\right)$$
To achieve scale invariance across different types of logs (e.g., thousands of web clicks vs. single digits of USB copies), we derive the final Contextual Z-Score Deviation ($z_t$):
$$z_t = \frac{x_t - \mu_{t}(x)}{\sqrt{\sigma^2_{t}(x)} + \epsilon}$$
Variable Annotations
* $x_t$: The raw feature count observed on day $t$ (e.g., total logins, file touches).
* $\lambda$: The exponential decay multiplier ($0 < \lambda \le 1$), determining how heavily recent behavior is weighted against past history. For a $14$-day span, $\lambda \approx 0.133$.
* $\mu_{t}(x)$: The calculated historical baseline mean for a specific user up to day $t$.
* $\sigma^2_{t}(x)$: The updated historical variance tracking the user's day-to-day consistency.
* $z_t$: The final normalized feature value representing how many standard deviations the user's current behavior drifts from their personal historical norm.
* $\epsilon$: An empirical smoothing regularizer (set to $1.0$) that bounds the denominator to prevent division-by-zero errors when analyzing highly inactive users.


📐 Step 2: One-Class SVM Non-Linear Hyperplane (Pillar 3A)
Once the 13-dimensional scale-invariant vector $X$ is compiled, it is routed to a One-Class Support Vector Machine. This algorithm maps the observations into a high-dimensional feature space where it constructs a smooth, curved boundary around normal user data.
The Equations
The algorithm maps the input data vectors into a reproducing kernel Hilbert space (RKHS) via a mapping function $\Phi(X)$. It solves the following primal quadratic optimization problem to build the maximum-margin hyperplane separating normal points from the origin:
$$\min_{w, \xi, \rho} \frac{1}{2} \|w\|^2 + \frac{1}{\nu n}\sum_{i=1}^{n}\xi_i - \rho$$
Subject to the geometric boundary constraints:
$$\langle w, \Phi(X_i) \rangle \ge \rho - \xi_i, \quad \xi_i \ge 0 \quad \forall i \in \{1, \dots, n\}$$
Because computing $\Phi(X)$ explicitly is computationally expensive, we apply the Kernel Trick using a non-linear Radial Basis Function (RBF) to calculate the inner products directly in the lower-dimensional space:
$$K(X_i, X_j) = \langle \Phi(X_i), \Phi(X_j) \rangle = \exp\left(-\gamma \|X_i - X_j\|^2\right)$$
The resulting decision function maps any user-day to a binary anomaly flag ($f_{\text{OC-SVM}}$):
$$f_{\text{OC-SVM}}(X) = \text{sign}\left( \sum_{i=1}^{n} \alpha_i K(X_i, X) - \rho \right)$$
Variable Annotations
* $w$: The normal vector perpendicular to the separating hyperplane boundary in the high-dimensional space.
* $\xi_i$: Slack variables that allow specific training points to fall on the wrong side of the boundary, preventing overfitting on background system noise.
* $\rho$: The margin parameter representing the linear offset distance of the hyperplane from the origin.
* $\nu$ (nu): A hyperparameter establishing a strict upper bound on the fraction of training errors (system noise contamination) and a lower bound on the number of support vectors (configured at $0.01$ or $1\%$).
* $\gamma$ (gamma): The RBF kernel width parameter controlling the radius of influence for individual support vectors. Setting it to 'scale' uses $\gamma = \frac{1}{\text{n\_features} \times \text{variance}(X)}$.
* $\alpha_i$: The Lagrange multipliers derived during optimization, where support vectors satisfy $\alpha_i > 0$.


📐 Step 3: Isolation Forest + Extreme Value Theory (Pillar 3B)
Concurrently, the data is evaluated by an Isolation Forest. This algorithm measures how easily data points can be isolated using random orthogonal cuts. Rather than applying a fixed anomaly score threshold, we apply Extreme Value Theory (EVT) to fit a continuous statistical model over the extreme right tail of the score distribution.
The Equations
Let $s(X)$ be the raw anomaly score generated by the Isolation Forest. To isolate the most severe anomalies, we invert the scores ($m_i = -s(X_i)$) and apply the Peaks-Over-Threshold (POT) model. Any score exceeding an initial high quantile cutoff $u$ generates an excess value ($y = m - u$).
According to the Pickands-Balkema-de Haan theorem, these tail excesses follow a Generalized Pareto Distribution (GPD):
$$G_{\gamma, \beta}(y) = P(M - u \le y \mid M > u) = 1 - \left(1 + \frac{\gamma y}{\beta}\right)^{-\frac{1}{\gamma}}$$
By using maximum likelihood estimation to isolate the shape ($\gamma$) and scale ($\beta$) parameters, we derive the dynamic EVT Threat Horizon Threshold ($\zeta_\alpha$) at a specific strict risk probability target $\alpha$ (configured at $0.015$):
$$\zeta_\alpha = u + \frac{\beta}{\gamma}\left[\left(\frac{n}{N_u} \alpha \right)^{-\gamma} - 1\right]$$
The structural decision function ($f_{\text{iForest+EVT}}$) flags anomalies where the inverted score breaches this mathematical horizon line:
$$f_{\text{iForest+EVT}}(X) = \mathbb{I}\Big(-s(X) \ge \zeta_\alpha\Big)$$
Variable Annotations
* $s(X)$: The raw isolation score, bounded between $0$ (normal) and $1$ (highly anomalous).
* $u$: The threshold baseline parameter, set at the $99\text{th}$ quantile of the empirical score distribution.
* $y$: The tail excess magnitude representing how far an anomalous point extends past the threshold baseline.
* $\gamma$: The GPD shape parameter (also known as the extreme value index) dictating the tail's behavior. A value of $\gamma > 0$ indicates a heavy-tailed distribution typical of rare cyber attacks.
* $\beta$: The GPD scale parameter defining the dispersion and spread of the tail excesses.
* $n$: The total number of observations in the dataset ($330,452$ user-days).
* $N_u$: The exact count of observations that exceed the initial threshold baseline $u$.
* $\alpha$: The target risk probability parameter ($0.015$), defining the acceptable threshold for extreme anomalies.
* $\zeta_\alpha$: The final dynamically computed threshold limit. If an inverted isolation score passes $\zeta_\alpha$, it is classified as an extreme anomaly.


📐 Step 4: Consensus Ensemble Intersection Logic
To balance false alarms against detection accuracy, the architecture fuses the decision boundaries of both individual models using a logical intersection operator ($\cap$).
The Equations
An active user-day is only flagged as a verified security incident ($A_{\text{final}} = 1$) if both independent classifiers mark the point as a confirmed anomaly simultaneously:
$$A_{\text{final}} = f_{\text{OC-SVM}}(X) \land f_{\text{iForest+EVT}}(X)$$
Substituting the underlying decision functions maps the final criteria into a unified logical equation:
$$A_{\text{final}} = \mathbb{I}\left(\sum_{i=1}^{n} \alpha_i K(X_i, X) - \rho < 0\right) \times \mathbb{I}\Big(-s(X) \ge \zeta_\alpha\Big)$$
Variable Annotations
* $A_{\text{final}}$: The final binary output indicator ($\in \{0, 1\}$), where $1$ triggers an analyst alert.
* $\mathbb{I}(\cdot)$: An indicator function returning exactly $1$ if the interior condition is satisfied and $0$ otherwise.
* $\land$: The logical AND operator requiring unanimous agreement between both detection layers.
